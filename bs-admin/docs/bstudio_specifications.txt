revision_idについて
　各バージョンにはprivate_revision_idが存在する。
　revisiont_id が private_revisiont_id 以上のレコードは上位バージョンからは見えない。
　新規バージョン作成時に既存バージョンの中のMAX（つまりその時点での最新バージョン）のprivate_rebision_idがインクリメントされる。
　このことにより、旧バージョンの変更は上位バージョンに影響を与えることがない。
　新バージョンを削除（最新バージョンかつ、作業中バージョン、公開バージョンのどちらでもない場合）し、もう一度新バージョンを作成すると
　旧バージョンのrevision_idが再びインクリメントされる。このことにより、新バージョン作成後に行った旧バージョンでの修正は、新バージョンにも反映されるようになる。

■module/editor
 function regist

 セーブする前にバージョンをチェックし、変わっている場合はコンテンツIDを更新する。
　updateNodeファンクションで行う。
　バージョン削除があった場合、旧ファイルが存在する可能性があるがその場合は「だれかが更新している」ダイアログが表示される→しょうがない。

■module/resource
 function saveName

 ファイル名を変更した場合、通常はコンテンツIDは変更しない（画像ファイルはバージョン間で共有）
　但し、拡張子を変えた場合はファイルの拡張子も同時に変更するのでバージョンが違っている場合は、別ファイルを作成しコンテンツIDを変更する。
　サムネイルファイルも同時に更新する。
